如何使用git工具把本地文件传给github

Created2025-07-21|Updated2025-07-26|hexo维护类
|Post Views:

以下是完整的本地项目推送到 GitHub 的全流程，包含密钥配置、仓库初始化及推送指令，所有操作均基于 2025 年最新 Git 实践：

一、生成 SSH 密钥并配置 GitHub

  1. 生成密钥对

    在 Git Bash 或终端运行（替换为你的 GitHub 邮箱）：

    • 连续按 3 次回车，默认路径（~/.ssh/id_rsa）和空密码。
    • 生成两个文件：
      • 私钥：id_rsa（严禁分享）
      • 公钥：id_rsa.pub（需上传 GitHub）

  2. 将公钥添加到 GitHub

    • 复制公钥内容：

    • 登录 GitHub →

      Settings

      →

      SSH and GPG keys

      →

      New SSH key

      • Title：My Local PC
      • Key：粘贴复制的公钥内容 → Add SSH key

  3. 测试 SSH 连接

    • 首次连接会提示确认指纹，输入 yes
    • 成功显示：Hi [你的用户名]! You've successfully authenticated...

二、初始化本地仓库并提交代码

  1. 进入项目目录

  2. 初始化 Git 仓库

    • 生成 .git 隐藏文件夹（版本控制核心）

  3. 暂存并提交代码

三、创建 GitHub 远程仓库并推送

  1. 在 GitHub 新建仓库

    • 登录 GitHub → Repositories → New
    • 填写仓库名（如 my-project）→ Create repository
    • 记住仓库的 SSH 地址（如 git@github.com:yourname/my-project.git）

  2. 关联本地与远程仓库

  3. 首次推送代码（强制关联分支）

    • -u 参数将本地分支与远程分支关联，后续可直接 git push
